# Frontend UI/UX

Use this skill when the task is primarily about component design, layout, interaction quality, or frontend polish.

## Workflow

1. Inspect the existing visual language first.
2. Preserve the product's current design system when one exists.
3. When no design system exists, choose one clear visual direction instead of a generic default.
4. Build responsive, accessible interfaces.
5. Verify behavior on both desktop and mobile layouts when applicable.

## Suggested delegation

- `architect` for UI structure and navigation decisions
- `executor` for implementation and polish
- `debugger` for layout, interaction, and browser issues
